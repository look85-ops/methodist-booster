## 1. Контекст и цели

### О продукте

Methodist Booster — набор из 5 инструментов для L&D-методиста, продакт-методиста и фасилитатора. Все инструменты живут в одном веб-приложении и решают разные задачи рабочего цикла.

**Текущее состояние:** функциональный MVP с одностраничной структурой, 5 вкладок, экспериментальное оформление. Готовый функциональный код лежит в `methodist-booster/game.html`.

**Что нужно от дизайна:**
- Превратить «инженерный прототип» в продукт портфолио-качества
- Сделать так, чтобы за 10 секунд было понятно, что это и для кого
- Обеспечить единый визуальный язык для всех 5 инструментов
- Сохранить «семейство» с портфолио автора (`natalialnd.netlify.app`)

### Стратегическое позиционирование

**Сейчас (MVP):** «Швейцарский нож методиста» — единый сервис под одним пользователем (методист L&D).

**С заделом на будущее:** каждая страница инструмента должна работать как самостоятельная посадочная — на случай, если решим разделить на 5 отдельных микро-продуктов под зонтиком.

### Целевая аудитория

- Внутренние L&D-методисты в корпорациях
- Тренеры и фасилитаторы
- Продакт-методисты, проектирующие обучающие продукты
- HR-Business Partners, отвечающие за обучение
- Внешние L&D-консультанты

**Что важно про аудиторию:**
- Технические руководители 35–50 лет с опытом — без коучингового сюсюканья
- Уважают честность, не любят «продающие приёмы»
- Ценят методическое обоснование («почему именно так»)

## 2. Архитектура продукта

### Структура страниц

```
Главная (landing)
├── Hero
├── Зачем это (короткий блок)
└── 5 плашек-ссылок на инструменты

/ivanov — Страдания инженера Иванова (симулятор)
/activities — Активности (AI-генератор)
/cards — Карты (фасилитационные)
/audit — Аудит (диагностика L&D)
/vision — Видение (мастер видения продукта)

Footer (сквозной на всех страницах)
├── Копирайт
├── Ссылка на портфолио автора
└── (опц.) ссылка на репозиторий
```

С каждой страницы инструмента — возврат на главную (логотип/название в шапке или явный CTA «← Главная»).

### Шаблон страницы инструмента

Экран делится **пополам** (split layout):

| Левая половина (40–45%) | Правая половина (55–60%) |
|---|---|
| Цветной/градиентный блок | Светлый, нейтральный |
| Название инструмента | Интерактивная зона |
| Зачем это / для чего | Поля ввода, кнопки |
| Как пользоваться (3–5 шагов) | Результат / ответ |
| Методическая основа | Скриншот «живого» состояния |
| (опц.) ссылка на статью | |

На мобильном — стек: сначала описание, потом интерфейс.

## 3. Визуальная система

### 3.1 Семейство с портфолио

**Сохраняем (узнаваемость):**
- Тёплый светлый фон (#FAF9F7)
- Шрифт Inter (Google Fonts)
- Радиальные градиенты как «фоновые пятна»
- Steel blue #3A5A78 как основной акцент
- Mint #22C1A0 как вторичный

**Усиливаем (Наталье «блекло» в текущем портфолио):**
- Градиенты ярче: alpha 30–45% вместо 16–18%
- Добавить третий тёплый акцент (терракот / коралл) для важных CTA
- Сильные цветные пятна на главной (hero) — большие, насыщенные
- Контрастные карточки/плашки (не выцветшие)

### 3.2 Палитра

**Основа:**
```
--bg:           #FAF9F7   тёплый светлый фон
--bg-elev:      #FFFFFF   поверхности карточек
--bg-dark:      #0F172A   тёмный текст / тёмные блоки
--text:         #0F172A
--text-muted:   #475569
--border:       #E5E7EB
```

**Акценты — оставляем семью + усиливаем:**
```
--accent-blue:  #3A5A78   главный (заголовки, ссылки, CTA)
--accent-blue-deep: #1F3B54   тёмный hover
--accent-mint:  #22C1A0   секонд (hover-рамки, badges, иконки)
--accent-mint-deep: #178A75   тёмный mint
--accent-coral: #D49C6C   новый тёплый акцент (CTA, важные плашки)
--accent-rose:  #F6A9C5   градиенты, мягкие плашки
```

**Тематика для 5 инструментов (каждый — свой цвет):**

| Инструмент | Цвет | HEX | Применение |
|---|---|---|---|
| Иванов | Mint | #22C1A0 | левая половина + бэйдж |
| Активности | Coral | #D49C6C | левая половина + бэйдж |
| Карты | Slate | #3A5A78 | левая половина + бэйдж |
| Аудит | Deep Rose | #B35E3A | левая половина + бэйдж |
| Видение | Purple-Blue | #6D5CE7 | левая половина + бэйдж |

Это даёт визуальную навигацию: с любой страницы видно, в каком инструменте ты сейчас.

### 3.3 Градиенты

**Hero на главной:** большое радиальное пятно из mint+coral+blue, alpha до 40% в центре, плавно гаснущее к краям. Размер пятна — половина экрана.

**Левые половины страниц-инструментов:** градиент из основного цвета инструмента → светлее на 30%, диагональный. Поверх — тонкий нойз (5–8%) для текстуры.

**Фон страниц:** тонкие радиальные пятна (как в портфолио, но 30% alpha вместо 18%).

### 3.4 Типографика

**Шрифт:** Inter (Google Fonts) — оставляем для семьи.

**Размеры:**
- H1 (hero главной): clamp(40px, 5vw, 64px), weight 700, letter-spacing -0.5px
- H1 (инструмент): clamp(32px, 4vw, 48px), weight 700
- H2: clamp(24px, 3vw, 36px), weight 700
- H3: 20px / weight 600
- Body large: 18px / weight 400, line-height 1.6
- Body: 16px / weight 400, line-height 1.6
- Small / meta: 14px / weight 500
- Micro / label: 12px / weight 600, letter-spacing 0.5px, uppercase

**Альтернатива для усиления:** допустимо использовать одного «компаньона» для крупных заголовков hero — например, **Manrope**, **Plus Jakarta Sans** или **Space Grotesk** — на твой выбор, если решишь добавить характер. Если сомневаешься — оставляй Inter везде.

### 3.5 Сетка и отступы

- Max-width контента: **1200px** (на десктопе с воздухом)
- Контейнер: max-width 1080px, padding 0 24px
- Отступы: 4 / 8 / 16 / 24 / 32 / 48 / 64 / 96
- Gap между секциями главной: 80–96px
- Gap внутри секций: 24–32px

### 3.6 Компоненты

#### Кнопки

**Primary** — `--accent-blue` фон, белый текст, padding 12px 24px, радиус 8px, font-weight 600. Hover — тёмный hover-цвет + лёгкий transform translateY(-1px).

**Secondary** — прозрачный фон, `--accent-blue` рамка 1.5px, цвет текста `--accent-blue`. Hover — заливка `--accent-blue` 8%.

**Ghost** — без рамки, только текст со стрелкой `→`. Hover — подчёркивание.

**Accent (важный CTA)** — `--accent-coral` фон, белый текст. Используется один раз на странице — например, «Попробовать сейчас».

#### Карточки

- Радиус 16px (мягче, чем в портфолио)
- Border 1px `--border`
- Тень: только при hover (0 8px 24px rgba(15,23,42,0.08))
- Padding 24–32px
- Hover: border меняется на цвет инструмента + transform translateY(-2px)

#### Чипы / теги

- Радиус 999px
- Padding 4–8px / 12px
- Border 1px
- Активный чип — заливка цветом инструмента + белый текст
- Неактивный — белый фон + `--border` рамка + `--text-muted` текст

#### Поля ввода / textarea

- Радиус 8px
- Border 1.5px `--border`
- Padding 12px 16px
- Focus — рамка цвета инструмента, легкая тень `0 0 0 3px [color]20%`
- Placeholder `--text-muted`

#### Прогресс-бары (для Иванова)

- Высота 12px, радиус 999px
- Фон `--border`
- Заливка — градиент от светлого к насыщенному в цвете шкалы (энергия/знания/перенос)
- Над баром — подпись и значение справа

#### Таймлайн (для Иванова)

- Высота 28px, радиус 999px, фон `--border`
- Сегменты — заливка цветом категории, белый текст внутри
- Подписи времени снизу (09:00, 12:00, 15:00, 18:00)

### 3.7 Иконография

**Подход:** нейтральные иконки (Lucide / Heroicons) + смысловые эмодзи там, где они несут смысл.

- **Эмодзи оставляем** в: настроении Иванова (😄 🙂 😕 😫 😵‍💫), вердиктах аудита (✅ ⚙️ 🚫), индикаторах состояния
- **Заменяем на иконки** в: пунктах меню, кнопках действий, заголовках секций, плашках инструментов

Иконки — 24x24, stroke 1.5, цвет inherit от родителя.

### 4.3 Контент по инструментам

#### 🎭 Страдания инженера Иванова

- **Цвет:** Mint #22C1A0
- **Слоган:** «Симулятор учебного дня. Проверь свой сценарий на стрессоустойчивость инженера»
- **Зачем:** Прежде чем рисковать на живой группе, прогони сценарий на симуляции. Видно сразу — где Иванов выгорает, где скучает, где наоборот включается.
- **Как пользоваться:**
  1. Выбирай форматы по таймлайну рабочего дня (9:00–18:00)
  2. Следи за 3 шкалами: энергия, знания, готовность применить
  3. Не забывай про обед и кофе-брейки — это часть методики, а не «провисание»
  4. В конце дня — финальный дашборд с разбором
- **Методическая основа:** Андрагогика, когнитивная нагрузка, Колб (опыт → рефлексия → концептуализация → применение)
- **Связанное:** Активности (если нужен формат для слота), Аудит (если непонятно, нужно ли вообще обучение)

#### ⚡ Активности

- **Цвет:** Coral #D49C6C
- **Слоган:** «Генератор активностей под твою тему. Без банальностей и «спорьте, почему 2+2=5»»
- **Зачем:** Когда нужна свежая активность для разогрева, кейс-разбора или закрытия модуля — здесь подбирается под параметры твоей сессии. Принципы andragogy зашиты в промпт, никаких «детских» механик.
- **Как пользоваться:**
  1. Укажи тему, цель (включение / осмысление / решение), формат, время
  2. Нажми «Сгенерировать»
  3. AI выдаёт структурированную активность: Название / Цель / Ход / Материалы / На что смотреть
  4. Не нравится — жми ещё раз, повторов не будет
- **Методическая основа:** Принципы andragogy, базовый каталог проверенных форматов
- **Связанное:** Иванов (вставить активность в день), Карты (если нужен вопрос для активности)

#### 🎴 Карты

- **Цвет:** Slate #3A5A78
- **Слоган:** «Колода фасилитационных вопросов. Для рефлексии, ретро, командных встреч»
- **Зачем:** В любой встрече бывает момент «нужен сильный вопрос, чтобы группа задумалась». Здесь — 29 проверенных вопросов в 5 категориях. К каждой карте — что слушать ведущему.
- **Как пользоваться:**
  1. Выбери категорию (старт / разбор / действие / напряжение / заказчик)
  2. Укажи размер группы — инструкция «Как провести» поменяется
  3. Вытяни карту. Не нравится — тяни ещё, повторов не будет
- **Методическая основа:** Coaching cards, фасилитационные практики, рефлексивные модели
- **Связанное:** Видение (если карты в начале vision-сессии), Иванов (если карта — основа активности)

#### 🔍 Аудит

- **Цвет:** Deep Rose #B35E3A
- **Слоган:** «Диагностика «обучение или не-обучение». Подготовка к разговору с заказчиком»
- **Зачем:** Бизнес часто приходит с запросом «научите наших». Но иногда обучение не лечит — нужна стратсессия, изменение процесса или работа с руководителями. Здесь — диагностика по Mager & Pipe и готовый бриф для заказчика.
- **Как пользоваться:**
  1. Опиши ситуацию и запрос заказчика своими словами
  2. Укажи аудиторию, цель, тип результата
  3. Выбери основную причину разрыва (6 вариантов с подсказками)
  4. Зафиксируй ограничения: объём, бюджет, людей, срочность, формат
  5. Получи вердикт + форматы + вопросы заказчику + риски
- **Методическая основа:** Mager & Pipe («Analyzing Performance Problems»), Kirkpatrick, organizational development
- **Связанное:** Видение (если задача требует продукта), Иванов (если решение — обучение)

#### 🎯 Видение

- **Цвет:** Purple-Blue #6D5CE7
- **Слоган:** «Vision-мастер для продакта. От боли пользователя — к минимально жизнеспособному решению»
- **Зачем:** Перед тем как делать продукт (курс, сервис, программу) — собери видение по строгой логике: боль → пользователь → job → альтернативы → решение → MVP → критерий. С методическим зеркалом, которое подсветит дыры.
- **Как пользоваться:**
  1. Пройди 7 шагов с наводящими вопросами
  2. На каждом шаге — «зачем мы это спрашиваем»
  3. На финале — саммари + проверка на типичные дыры + 3–5 направлений решения
  4. Можно скопировать как 1-страничный бриф
- **Методическая основа:** Lean Startup, Jobs-to-be-Done, Product Vision frameworks
- **Связанное:** Аудит (если ещё не уверены, обучение или нет), Активности (если в скоупе курс — наполнение модулей)

## 5. Адаптивность

### Брейкпоинты

- Desktop large: ≥1280px
- Desktop: 1024–1279px
- Tablet: 768–1023px
- Mobile: <768px

### Поведение

- **Главная:** 5 плашек — 2-3-2 на десктопе, 2-2-1 на планшете, 1-в-колонку на мобильном
- **Страницы инструментов:** split 42/58 на ≥1024px, стек на <1024px (сначала левая колонка с описанием, потом правая с интерактивом)
- **Header:** меню скрывается в гамбургер на <768px
- **Hero:** уменьшается, иллюстрация прячется на <768px
- **Таймлайн Иванова:** горизонтальный скролл, если не помещается

---

## 6. Доступность

- Контраст текст/фон ≥ 4.5:1 (WCAG AA)
- Контраст крупных заголовков ≥ 3:1
- Все интерактивные элементы достижимы клавиатурой (Tab/Enter/Esc)
- Видимый focus-ring (2px outline в цвете инструмента + offset 2px)
- Aria-labels для иконок-кнопок
- Alt для иллюстраций (если будут)
- Размер кликабельной зоны ≥ 44x44px

## 7. Анимации и микровзаимодействия

**Принцип:** минимум, но точечно.

- **Hover карточек:** transform translateY(-2px) + смена border-color + лёгкая тень (transition 200ms ease)
- **Кнопки:** transform translateY(-1px) на hover, scale(0.98) на active
- **Появление результата:** fade-in 300ms + slide-up 8px
- **Шкалы Иванова:** анимация ширины 300ms ease при изменении
- **Таймлайн:** новый сегмент появляется с fade-in 200ms
- **Микро-обратная связь:** на копировании — плашка «Скопировано» на 1.5 сек

**Чего НЕ делаем:**
- Parallax-эффектов
- Анимаций при скролле
- Сложных переходов между страницами

## 8. Иллюстрации и графика

**Что нужно нарисовать:**

1. **Иллюстрация для hero главной** — абстрактная композиция из 5 элементов (по числу инструментов). Стиль: плоская, мягкие формы, градиенты, может быть линиями. НЕ фотореализм, НЕ 3D, НЕ детское.

2. **5 иконок-маркеров инструментов** — упрощённые символы, по 1 на инструмент. Стиль одинаковый, размер ~64x64.

3. **Опциональные декоративные элементы**: волнистые линии, геометрические формы для разделения секций.

**Чего НЕ берём:**
- Стоковые фото людей
- Микс растровых и векторных стилей
- Слишком детальные иллюстрации, которые отвлекают от текста

## 10. Ограничения и принципы

**DO:**
- Тёплый светлый фон, как в портфолио

- Доступность по умолчанию (контраст, фокус, клавиатура)

- Тёмная тема (не делаем — только светлая)

- Полностью переходить на другой шрифт (теряем семью)
- Использовать стоковые фото
- Делать «сложный» интерактив (карусели, аккордеоны без причины)
- Перегружать иконками и эмодзи

## 13. Зафиксированные решения

1. **Адрес сайта:** отдельный поддомен `booster.natalialnd.com` (или эквивалентный отдельный netlify-домен — настроить при старте).

   - 1 hero-композиция на главной (большая, из 5 цветных форм по числу инструментов)

   - Опциональные декоративные элементы (волнистые линии, геометрические формы для разделения секций)
   - Все цвета — из дизайн-токенов, чтобы при изменении палитры иллюстрации обновлялись автоматически
3. **Тема:** только светлая (как в текущем портфолио). Тёмную тему не делаем даже опционально.
4. **Локализация:** только русский. Структуру под i18n не закладываем.
5. **Аналитика:** не подключаем.
6. **Форма обратной связи:** не делаем.
