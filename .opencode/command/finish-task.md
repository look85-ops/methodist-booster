# /finish-task

Закрыть текущую задачу, выполнить ритуал обучения и зафиксировать результат. Пользователь не вносит правки вручную: ассистент сам обновляет файлы и выводит итог с рекомендацией.

## Процесс

1. Проверить критерии готовности: пройтись по чекбоксам в описании фичи.
2. Если все критерии выполнены → подготовить статус к обновлению на "Готово" в `docs/BACKLOG.md` и дату закрытия. Если нет → отметить, что требуется доработка, и перечислить недостающие пункты.
3. Зафиксировать результат (кратко: что сделано, риски/заметки).
4. Ритуал обучения (автоматически обновить файлы):
   - Сохранить 1–3 инсайта в `knowledge/` (linkedin-rules, career-insights, process-lessons)
   - Добавить строку в `logs/decisions.md`: `F-NNN: что выбрали — почему`
   - Проверить уровень автономии для этого типа действий и предложить повысить/понизить при необходимости
5. Подготовить коммит: показать diff и готовое сообщение. По умолчанию предложить подтвердить коммит одной командой. Формат: `[F-NNN] Название фичи`.
6. Если работаем не в main — предложить создать PR в `main` и дождаться зелёного CI.
7. Предложить следующую приоритетную задачу: «Берём F-NNN?»

## Когда вызывать

Когда пользователь говорит: «Закрой задачу», «Готово, коммить», «Закончили, коммитим», фича выполнена.

## Формат коммита

```
[F-NNN] Название фичи

- Что сделано 1
- Что сделано 2
```

## Метрики и последующее наблюдение

- Для постов: через 7 дней запросить у пользователя метрики (просмотры/реакции/комментарии) и дописать вердикт в `logs/decisions.md` + инсайт в `knowledge/linkedin-rules.md`.
- Для резюме/откликов: через 7–14 дней запросить количество откликов/интервью и зафиксировать в `logs/decisions.md` + при необходимости — в `knowledge/career-insights.md`.

## Выход ассистента (формат ответа пользователю)

Ассистент выводит краткий отчёт в одном сообщении без необходимости ручных правок:

1) Итог по фиче (1–3 тезиса)
2) Чек критериев готовности:
   - [x] Пункт A
   - [ ] Пункт B (что мешает)
3) Изменения в файлах (что обновлено):
   - knowledge/...: короткая цитата добавленного пункта
   - logs/decisions.md: показываем строку F-NNN: ...
4) Рекомендация: Завершить | Доработать
   - Если «Доработать»: 1–3 конкретных шага с оценкой времени
5) Автономность: предложение по уровню (🟢/🟡/🔴) и основание
6) Follow-up: напоминания по метрикам (дата, что собрать)
