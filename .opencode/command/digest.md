# /digest

Запускает производство выпуска «L&D × AI Дайджест»: research через websearch/webfetch → синтез → публикация.

## Процесс

### 1. Выбор темы
- Проверяем backlog идей (если есть)
- Фиксируем фокус выпуска

### 2. Research (webfetch DuckDuckGo Lite)
Выполняем параллельные поиски через `webfetch` по URL:
```
https://lite.duckduckgo.com/lite/?q={query}&num=8
```

Стандартные запросы (адаптировать под тему выпуска):
- AI corporate training + год
- learning analytics AI L&D + год
- instructional design AI tools + год
- edtech AI product trends + год

Используем 3-4 параллельных запроса (максимум — DDG начинает капчить).

После получения результатов — выборочно `webfetch` 1-2 ключевых статьи для деталей.

### 3. Синтез
Из raw-результатов отбираем:
- **Кейс недели** — история/отчёт с цифрами (что сделали, как мерили, результат)
- **Инструмент** — 1 инструмент под 1 конкретную задачу
- **Метрика/фреймворк** — 1 показатель или подход
- **3 ссылки** — каждая с 1 строкой «почему важно»

Правила отбора:
- Каждый элемент отвечает на вопрос «что с этим делать в понедельник»
- Один выпуск — одна сквозная тема
- Язык: RU, без англицизмов там, где есть русский аналог
- Все цифры — из источника, без «по оценкам экспертов»

### 4. Пост-райтер
- Прогоняем через humanizer-правила (29 паттернов)
- Проверяем CTA
- Записываем в `career/posts/digest-{YYYY-MM-DD}.md` + `.html`

### 5. Публикация
- Показываем пользователю финальную версию
- После подтверждения — публикуем в LinkedIn
- Добавляем задачу на замер метрик через 7 дней

### 6. После публикации
- Записать 1-2 инсайта в `knowledge/linkedin-rules.md`
- Добавить строку в `logs/decisions.md`

## Структура поста (LinkedIn)

Заголовок: первая строка — крючок (вопрос, цифра, факт)
↓
Кейс недели (120-180 слов)
↓
Инструмент (40-60 слов + ссылка)
↓
Метрика/фреймворк (40-60 слов)
↓
3 ссылки (3 строки с «почему важно»)
↓
Рекомендация (конкретное действие)
↓
CTA + анонс следующего выпуска

## Параметры

- `--topic` — тема выпуска (если не указана — выбирается автоматически)
- `--dry-run` — только research, без записи и публикации

## Когда вызывать

- Раз в 2 недели (каждый второй понедельник)
- По команде пользователя: «выпусти дайджест», «сделай дайджест», `/digest`

## Guardrails

- Не публиковать без подтверждения пользователем
- Если DDG заблокировал — пауза 30 сек, повтор с другим запросом
- Не использовать непроверенные данные — все цифры должны быть из источника
