# /ralph — План задач и валидация по PRD

Назначение: превратить утверждённый PRD в компактный план задач с точками валидации результата.

Вход:
- Ссылка на PRD (`docs/prd/PRD-*.md`)

Выход:
- Файл плана в `docs/prd/PLAN-YYYY-MM-DD-<slug>.md` по шаблону `TASK_PLAN_TEMPLATE.md`

Как работает ассистент при вызове `/ralph`:
1. Извлекает ключевые результаты/КПЭ из PRD.
2. Декомпозирует на 5–15 задач с краткими описаниями.
3. Назначает точки валидации: что и как проверяется (тест/прогон/опрос/сборка/релиз‑артефакт).
4. Отмечает риски и допущения; помечает «блокеры».
5. Возвращает путь к файлу и короткий road‑checklist перед стартом.

Шаблон: `docs/prd/TASK_PLAN_TEMPLATE.md`
