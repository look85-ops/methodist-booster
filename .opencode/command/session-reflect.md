# /session-reflect

Анализирует текущую сессию и `logs/decisions.md`, извлекает уроки, паттерны, идеи для улучшений. Формирует рекомендации по записи в память и бэклог.

## Процесс

1. **Прочитать `logs/decisions.md`** — все записи решений
2. **Проанализировать текущую сессию:** что делали, какие файлы меняли, какие решения приняли, что не сработало
3. **Сформировать отчёт:**
   - 3–5 наблюдений (повторяющиеся паттерны, узкие места, удачные приёмы)
   - 1–2 действия (что улучшить в процессе или инструментах)
   - Предложения по записи в `knowledge/` (какой файл, что добавить)
   - Идеи для новых задач в `docs/BACKLOG.md`
4. **Спросить пользователя:** что из этого записать в память/бэклог
5. **По подтверждению:** записать инсайты в `knowledge/*` и/или добавить задачи в `docs/BACKLOG.md`

## Когда вызывать

- После завершения крупной задачи
- Когда пользователь говорит «подведи итоги сессии», «рефлексия», «что мы сделали»
- Раз в 2 недели как рутина

## Формат отчёта

### Наблюдения
1. ... (что заметили)
2. ... (что заметили)

### Действия
- [ ] ... (что улучшить)

### В память
- `knowledge/...` — что добавить

### В бэклог
- Идея для задачи: ...

## Guardrails

- Не предлагать больше 5 наблюдений и 2 действий — воронка, а не свалка
- Каждое наблющение должно подтверждаться фактом (ссылка на файл, строку, событие)
- Не записывать в память без подтверждения пользователя
