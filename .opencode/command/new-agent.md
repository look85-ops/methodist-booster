# /new-agent

Создаёт нового субагента и регистрирует его в системе.

## Аргументы

| Аргумент | Обязательный | Описание |
|----------|--------------|----------|
| `name`   | Да           | Имя агента (snake_case, например `career-architect`) |

## Сценарий

### Шаг 1: Изучить контекст проекта
Прочитай файл `AGENTS.md` в корне рабочей директории. Пойми:
- Существующих агентов и их зоны ответственности
- Структуру документа
- Стиль описаний

### Шаг 2: Уточнить назначение
Спроси у пользователя:
- Зачем тебе этот агент?
- Что конкретно он должен делать?
- В каких ситуациях его вызывать?

### Шаг 3: Проверить пересечения
Если видно, что новый агент пересекается с существующими:
1. Подсвети эти места
2. Предложи вариант решения:
   - Объединить агентов
   - Сузить зону ответственности нового
   - Оставить оба с разными фокусами
3. Дождись решения пользователя

### Шаг 4: Дополнительные вопросы
Уточни:
- **Разрешения**: может ли агент править файлы, запускать команды?
- **Формат ответа**: что агент должен возвращать?
- **Контекст**: какие файлы/данные агенту нужны для работы?

### Шаг 5: Предложить улучшения
Исходя из ответов, предложи способы усилить бриф:
- Добавь типичные сценарии использования
- Предложи инструменты, которых нет в брифе
- Уточни граничные условия

### Шаг 6: Финальный бриф
Покажи пользователю черновик брифа в формате:

```
## [Category] Agent Name

**Назначение:** ...
**Разрешения:** ...
**Формат ответа:** ...
**Зоны ответственности:**
- ...
**Инструменты:** ...
**Связанные агенты:** ...
```

Жди подтверждения (`ОК`, `Всё верно` или аналог).

### Шаг 7: Создать файл агента
Сгенерируй файл `.opencode/agent/<name>.md` с полным брифом.

### Шаг 8: Зарегистрировать агента
Создай общий командный файл в `.opencode/command/` со структурой:
```markdown
# /<command-name>

<Краткое описание команды>

## Аргументы
...

## Сценарий
<Ссылка на агента или inline-описание>
```

Обнови таблицу **Roles** в `AGENTS.md`, добавив нового агента.

### Шаг 9: Отчёт
Покажи пользователю итоговый отчёт:
- Путь к файлу агента
- Путь к командному файлу
- Что добавлено в AGENTS.md
- Как протестировать

## Правила

1. **Всегда жди подтверждения** перед созданием файлов
2. **Не угадывай** непонятные моменты — задавай вопросы
3. **Пересечения — это нормально**, но требуют явного решения
4. **Имя агента** — snake_case, без пробелов и спецсимволов
5. **Формат файла агента** — стандартный markdown с секциями
