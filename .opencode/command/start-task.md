# /start-task

Взять фичу из бэклога и начать разработку.

## Процесс

1. **Найти фичу** по ID F-NNN в `docs/BACKLOG.md`

2. **Проверить архитектурные решения:**
   - Если фича новая и сложная → вызвать `product-analyst` для архитектурного дизайна
   - Спросить: "Нужно ли проектирование? Что уже есть?"
   - Если нужны решения → задокументировать их перед началом
   - Для Course Ops (сценарии «Профессионал +. Путь»/«Академия пути») → активировать навык `learning-designer` (см. `.opencode/agent/learning-designer.md`) и спланировать день через его паттерны (вызовы → теория → применение; сквозной фреймворк; цикл рефлексии)

3. **Обновить статус** фичи на "В работе"

4. **Создать план работ:**
   - Разбить на подзадачи (если большая фича)
   - Определить порядок действий
   - Записать в конец описания фичи
   - Для Course Ops: зафиксировать выбранный сквозной фреймворк (напр., RACI) и точки включения «вызовов»

5. **Сообщить пользователю:**
   - Что делаем
   - Сколько подзадач
   - С чего начинаем

## Когда вызывать

Когда пользователь говорит:
- "Начни задачу F-001"
- "Возьми фичу..."
- "Работаем над..."

## Параметры

- `F-NNN` — ID фичи из BACKLOG.md (например, F-001, F-042)
