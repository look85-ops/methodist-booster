# /prd — Шаблон и протокол PRD

Назначение: превратить размытые требования/идею в короткий, прикладной PRD (Product Requirements Document) по единому шаблону.

Вход:
- Короткое описание идеи/запроса (2–5 предложений)
- Контекст (ссылки на материалы, метрики, ограничения)

Выход:
- Файл PRD в `docs/prd/PRD-YYYY-MM-DD-<slug>.md` по шаблону `PRD_TEMPLATE.md`

Как работает ассистент при вызове `/prd`:
1. Уточняет недостающие данные по пунктам шаблона (минимально).
2. Создаёт файл из шаблона, заполняет известные поля, помечает пропуски `Недостающий материал: …`.
3. Фиксирует критерии успеха и метрики (если есть АОС — ссылка на SOP и отчёты).
4. Возвращает путь к файлу и чек‑лист согласования PRD.

Шаблон: `docs/prd/PRD_TEMPLATE.md`

Опционально после `/prd`:
- `/ralph` — сформировать план задач и точки валидации по PRD
