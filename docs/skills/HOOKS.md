# Automation Hooks (токен‑экономия)

Два лёгких хука для снижения токенов и повышения ясности.

## 1) Prompt length linter

Триггер: пользовательский запрос > 50 слов.

Действие: дописать короткую заметку к промпту: «Пожалуйста, проверь, что желаемый результат сформулирован явно (1–2 предложения)».

Отключение: `HOOK_PROMPT_LINTER=0`.

## 2) Optimization hint on heavy tool use

Триггер: ≥ 8 вызовов инструментов в рамках одной сессии.

Действие: дописать одну фразу‑совет (реиспользование skill, память, workflow).

Отключение: `HOOK_OPT_HINT=0`.

## Встраивание

- Claude Code/Cursor: реализовать как UserPromptSubmit hook в глобальных настройках/скрипте окружения.
- Сторонние плагины: аналогичные события на сабмите запроса.

## Псевдокод

```
onUserPromptSubmit(prompt, session) {
  if (env.HOOK_PROMPT_LINTER != '0' && wordCount(prompt) > 50) {
    prompt += "\n\n[Note] Проверьте, что желаемый результат сформулирован явно.";
  }
  if (env.HOOK_OPT_HINT != '0' && session.toolCalls >= 8) {
    prompt += "\n\n[Hint] Рассмотрите: вынести повторяемые шаги в skill или сохранить промежуточные артефакты в память.";
  }
  return prompt;
}
```
