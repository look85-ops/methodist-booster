# Стандарт: динамическая инъекция контекста в skills

Цель: подставлять актуальные данные (вывод CLI/скриптов) в промпт перед отправкой в LLM.

## Синтаксис

- Маркер выполнения: `!<command>`
- Пример:
  - PR diff: `!gh pr diff`
  - Комментарии: `!gh pr view --comments`
  - Изменённые файлы: `!gh pr diff --name-only`

Вывод команды заменяет плейсхолдер в тексте skill. LLM получает данные, а не команду.

## Правила безопасности (см. SHELL_GUARDS)

- Включён allowlist команд (по умолчанию: `gh`, `git`, безопасные утилиты чтения)
- Ограничение объёма вывода (обрезка до N КБ + указание, что обрезано)
- Редакция чувствительных данных (токены, ключи)
- Логирование вызовов (команда, длительность, объём вывода)
- Флаг `disableSkillShellExecution` для отключения выполнения

## Мини-шаблон skill с инъекцией

```
---
name: pr-summary
description: Summarize changes in a pull request
context: fork
agent: Explore
allowed-tools: Bash(gh *)
---

## Pull request context

- PR diff: `!gh pr diff`
- PR comments: `!gh pr view --comments`
- Changed files: `!gh pr diff --name-only`

## Your task

Summarize this pull request…
```

## Когда использовать

- PR/Issue обзоры, генерация release notes
- Сбор данных из внешних API/CLI, которые дешевле/точнее, чем «вспоминать» в модели

## Когда не использовать

- Команды, требующие побочных эффектов (запись, удаление)
- Команды с потенциальной утечкой секретов

## Отладка

- Запускайте команды вручную, если нужно сверить объём и формат
- Включайте `--json` где возможно для структурного вывода
