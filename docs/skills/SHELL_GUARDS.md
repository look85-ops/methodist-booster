# Guardrails для shell‑инъекций

Цель: безопасно выполнять `!<command>` внутри skills.

## Политика

- Allowlist по бинарям: `gh`, `git`, `python -c` (только чтение), `type`, `dir`.
- Запрет на команды записи/удаления: `rm`, `del`, `mv`, `cp`, `curl -X POST`, и т.п.
- Лимит вывода: по умолчанию 64 КБ, далее обрезка с пометкой.
- Тайм‑аут: 10 секунд на команду.
- Маскирование чувствительных паттернов: ключи, токены, `Authorization:`.
- Флаг `disableSkillShellExecution` — глобально выключает выполнение.

## Шаблон обработки

1. Проверить включённость флага.
2. Провалидировать команду по allowlist.
3. Выполнить с тайм‑аутом и лимитом вывода.
4. Отредактировать чувствительные данные.
5. Подставить результат в промпт.

## Пример allowlist

```
allowed = [
  /^gh\s+pr\s+(diff|view)(\s+--[\w-]+)*/,
  /^git\s+status/, /^git\s+diff\b/, /^git\s+log\b/,
  /^type\b/, /^dir\b/
]
```

## Отказ/ошибки

- Неизвестная команда → заменить плейсхолдер текстом: «[EXECUTION DISABLED/NOT ALLOWED]».
- Тайм‑аут → «[OUTPUT TRUNCATED: TIMEOUT]».
- Обрезка → «[OUTPUT TRUNCATED: SIZE LIMIT]».
