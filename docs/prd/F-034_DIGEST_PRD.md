# PRD: L&D × AI Дайджест

Дата: 2026-06-08
Автор: /digest
Статус: Draft

## 1. Контекст и проблема

L&D-специалисты, методологи и edtech-менеджеры тонут в информационном шуме: десятки телеграм-каналов, рассылок, новостей про AI. Большинство дайджестов — просто подборки ссылок без выводов «что с этим делать».

Проблема: нет дайджеста, который:
- Отбирает сигналы из сотен источников (а не 2-3 телеграм-канала)
- Даёт не «что вышло», а «что с этим делать в понедельник»
- Учитывает три аудитории: менеджеры проектов, методологи, edtech

## 2. Цель и гипотеза

**Цель:** Стать регулярным (раз в 2 недели) профессиональным дайджестом №1 для русскоязычных L&D × AI-специалистов.

**Гипотеза ценности:** Если дайджест даёт готовое к применению знание (кейс + инструмент + метрика), а не просто новости, аудитория будет возвращаться и шерить.

## 3. Скоуп

**Входит:**
- Регулярный выпуск раз в 2 недели
- 4 блока: кейс, инструмент, метрика/фреймворк, 3 ссылки
- Исследование через last30days-skill + собственный поиск
- Публикация в LinkedIn
- Замер метрик через 7 дней после публикации

**Не входит:**
- Телеграм-канал или email-рассылка
- Платная реклама дайджеста
- Видео-формат

## 4. Требования к решению

**Сценарий производства:**
1. Выбор темы выпуска (на основе current topics + backlog идей)
2. Research: last30days по 3-5 ключевым запросам
3. Синтез: отбор сигналов по сетке «интересно × полезно для ЦА»
4. Написание по шаблону
5. Humanizer-чекаут (пост-райтер)
6. Публикация в LinkedIn
7. Через 7 дней — замер метрик + запись в knowledge

**Сетка тем (ротация):**
| Выпуск | Фокус |
|--------|-------|
| 1 | AI-инструменты для L&D |
| 2 | Метрики и оценка обучения |
| 3 | Проектирование программ с AI |
| 4 | Edtech-рынок: тренды |
| 5 | Кейсы внедрения |
| 6 | Фреймворки и методология |

## 5. Метрики успеха

- **Основные:** охват поста в LinkedIn > 500 к 3-му выпуску, ER > 3%
- **Вспомогательные:** количество сохранений, комментарии от ЦА

## 6. Риски

- last30days может не дать релевантных результатов по RU-темам — нужен fallback (веб-поиск ассистента)
- LinkedIn может не показывать длинные посты в ленте — заголовок должен зацеплять с первой строки
